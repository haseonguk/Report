# 🚗 스마트 차량 점검 리포트 (Smart Vehicle Reference Report) v3.5

정확한 기준 데이터(Reference)를 바탕으로 차량 점검 내역을 분석하고, 전문적인 PDF 리포트를 생성하며, 구글 시트를 통해 실시간으로 데이터를 관리하는 업무 자동화 솔루션입니다.

---

## ✨ 주요 기능

- **🎯 스마트 기준데이터 매칭 (Reference Matching)**: 
  - 단순 AI 생성 대신, 현장의 노하우가 담긴 **마스터 레퍼런스 데이터(구글 시트)**를 기반으로 작동합니다.
  - 사용자가 입력한 **차종, 증상, 주행거리**를 분석하여 가장 적합한 정비 항목과 권고사항을 자동으로 매칭하여 제안합니다.
- **📄 맞춤형 PDF 리포트 발행**: 매칭된 데이터를 바탕으로 깔끔한 디자인의 전문 점검 리포트를 PDF로 생성하며, 카카오톡/문자 공유를 지원합니다.
- **📱 PWA(Progressive Web App) 지원**: 별도의 앱 설치 없이 모바일/데스크톱 홈 화면에 추가하여 현장에서 즉시 리포트를 작성할 수 있습니다.
- **📊 리얼타임 데이터 저장**: 모든 발행 내역과 고객 정보가 구글 스프레드시트에 실시간으로 기록되어 별도의 DB 없이 관리가 용이합니다.
- **🏢 독립적 멀티 지점 지원**: 하나의 앱 엔진으로 여러 대표/지점이 각자의 독립된 구글 시트를 연결하여 사용하는 공유 경제형 구조입니다.

---

## 🛠 기술 및 로직 설명

### 🔍 기준데이터(Reference) 매칭 기술
본 프로젝트는 불확실한 AI의 답변 대신, 검증된 데이터를 매칭하는 **스코어링 알고리즘**을 사용합니다:
1. **차종 매칭**: 입력된 차종과 레퍼런스 차종의 유사도를 분석 (가장 높은 가중치).
2. **증상 키워드 분석**: 입력된 증상에서 정비 키워드를 추출하여 관련 데이터와 매칭.
3. **주행거리 근접도**: 현재 주행거리를 기준으로 정비 주기가 근접한 항목 우선 추천.

## 🛠 기술 스택

- **Frontend**: React (Vite), Tailwind CSS
- **Backend**: Google Apps Script (GAS)
- **Database**: Google Sheets (Infrastructure)
- **Deployment**: Vercel

---

## 🚀 시작하기

### 1. 프론트엔드 설정 (Frontend)
1. `frontend` 폴더에서 필요한 라이브러리를 설치합니다.
   ```bash
   cd frontend
   npm install
   ```
2. 로컬에서 실행합니다.
   ```bash
   npm run dev
   ```

### 2. 백엔드 설정 (Google Apps Script)
1. `backend` 폴더 내의 코드를 구글 앱스 스크립트 에디터에 복사합니다.
2. [배포] -> [새 배포] -> [웹 앱]으로 배포합니다 (액세스 권한: 모든 사용자).
3. 스프레드시트 트리거를 '설문지 제출 시'로 설정합니다.
